## Installing the Homebrew package defaultbrowser

In order to set my default browser in macOS via Command Line, I first had to
`install the Homebrew package called "defaultbrowser".

First, I ran the following commands, one after the other`:

```shell
brew update
brew upgrade
```

This is similar to what we do in Linux Mint when we run `sudo apt update` and
`sudo apt upgrade` to make sure that all our packages are up to date before
installing a new one. We have to do the same with Homebrew.

## Installing defaultbrowser with Homebrew

Next, I run the following command to install the defaultbrowser package:

```shell
brew install defaultbrowser
```

## Running the defaultbrowser command

After it is successfully installed, I quit the active Terminal window and open
up a new instance. And there, I run the following command to make Safari my
default browser:

```shell
defaultbrowser safari
```

Chances are that the `defaultbrowser` command will be able to change my default
web browser as long as the target browser is installed on my machine. Currently
I have the following browsers installed, and I can set any of them as my default
browser:

```shell
Firefox Developer Edition.app
Firefox Nightly.app
Firefox.app
Google Chrome.app
Microsoft Edge.app
Opera.app
Safari.app
Vivaldi.app
```

## Running defaultbrowser without any arguments

I can run the `defaultbrowser` command without any arguments, and it will return
all the browser apps installed on my machine. That is how I got the list of
browser apps included above. However, it will also list other apps that are not
purely browser apps but might contain some kind of browser functionality.

## Limitations of defaultbrowser

|                               | Limitations of defaultbrowser                                                                                                                                                                                             |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Compatibility Issues**      | **defaultbrowser** may not support all web browsers. You should verify if your preferred browser is compatible.                                                                                                           |
|                               | **defaultbrowser** may not be compatible with all versions of macOS.You should check if your macOS version is compatible with **defaultbrowser**.                                                                         |
| **User account restrictions** | **defaultbrowser** may not work well in environments with multiple user accounts. Homebrew's permission model can restrict access, making it difficult for you and other users on the system to set your default browser. |
| **Installation requirements** | You must have Homebrew installed to use **defaultbrowser**, which adds an extra step if you are unfamiliar with package managers and Command Line.                                                                        |
| **Command limitations**       | While the `defaultbrowser` command is simple, you still need to be familiar with Command Line, which could be an obstacle.                                                                                                |

## Running the defaultbrowser command with Monterey 12.3 and beyond

Monterey 12.3 introduced the requirement for users to confirm their desire to
switch their default browser from Safari to something else. Now, if I run the
`defaultbrowser` command in Terminal to switch away from Safari as my default
browser OR switch back to it, I receive something like the following prompt:

![macOS prompt requiring a user to confirm their desire to switch their default browser](Screenshot-2026-06-06-at-11.11.34-AM.jpg)

_Screenshot of macOS prompt requiring a user to confirm their desire to switch
their default browser_

This prompt appears whether I am using the `defaultbrowser` command in Terminal
or open a web browser which is NOT my default browser. It does not happen when I
run a shell script. In fact, nothing happens. When I open the browser I have set
to my default using the script, I still have to confirm the switch. And then I
also have to confirm the switch with macOS!

## defaultbrowser works because it does things the "right way"

defaultbrowser still works because it does things the "right way". It calls
`LSSetDefaultHandlerForURLScheme` in the Launch Services framework, which is the
official macOS API for setting a default browser. The confirmation dialog it
triggers is the correct dialog; it's the OS itself asking the user to confirm.
There's no scripting path that skips that step in modern macOS.

## Conclusion

In pre-Monterey macOS, scripting the setting of a default browser was possible
via plist edits or old System Preferences AppleScript. However, with Monterey
and after, the only legitimate programmatic path is `defaultbrowser` (or any
tool calling the same API), and the system dialog is unavoidable by Apple
design. In addition, most AppleScripts became obsolete with the release of macOS
Ventura. Therefore, if working inside the Command Line is your preference, the
`defaultbrowser` Homebrew package is the way to go.
